DingTalk solution review

钉钉投票方案调研摘要

评估官方能力、第三方应用与低代码平台，判断其能否满足两委会记名评审、权限隔离、改票留痕、催投和长期保存要求。

调研日期 2026-07-18
评估范围 官方、市场应用、低代码
核心原则 未公开证明即待验证
核心结论
1

建议采用自研 H5 + 后端

现有公开应用没有被证实能够同时满足权限隔离、完整改票历史、精准私聊催投、服务端关闭和长期可恢复备份。自研方案能够把全部规则落实为可验收、可审计的系统能力。

问卷星和宜搭保留为替代方案参考，不作为当前首选。若未来考虑采购，必须先通过五项 POC 门槛。

三条可执行路径

调研结果不是简单的买或不买。不同路径对应不同的审计强度、交付速度和运维责任。

建议决策 01

自研 H5 + 后端

直接按两委会评审规则建设，能够明确保证权限、服务器端截止、一人一票、追加式版本历史、私聊催投日志和每日可恢复备份，避免把关键审计要求建立在供应商未公开承诺的能力上。

  • 钉钉身份登录与委员名单校验
  • 仅 HR 查看记名明细
  • 投票期间完全隐藏结果
  • 截止前修改并保留完整版本
  • 一键私聊未投委员并留痕
  • 长期保存、每日备份和恢复验证
推荐结论 自研 以可控、可审计和严格满足业务规则为优先级
最接近成品的备选 02

问卷星企业尊享版

公开年费 ¥12,000/年，钉钉渠道实际报价需另行确认。公开资料覆盖多数通用能力，但完整且不可覆盖的改票版本链、精准钉钉私聊催投等关键要求仍需 POC 和合同确认。

低代码备选 03

宜搭专业版

权限、修改记录、定时触发和通知能力较齐全，但仍需设计数据模型、规则和外部备份。它减少编码，不消除实施与验收工作。

真正决定选型的三个风险

普通投票、统计和导出并不难。差异集中在审计、关闭控制和消息触达。

3

不可用宣传页代替验收

所有关键风险都必须用真实委员账号、真实改票动作和真实导出文件验证。

审计风险

改票是否覆盖旧值

“允许修改”不等于“保留历史”。必须确认每一版选择、意见、操作人和时间都可追溯。

权限风险

结果是否存在旁路

除了页面按钮，还要检查分享链接、统计页、报表、API 和平台管理员权限。

执行风险

关闭和催投是否可靠

截止必须由服务端强制执行。催投需要精确计算未投名单，逐人发送，并留下成功、失败和重复操作记录。

供应商 POC 验收门槛

自研 H5 为建议方案。问卷星企业尊享版公开价为 ¥12,000/年，因此只有以下门槛全部通过，才值得进入正式报价和合同评审。

  1. 连续改票留痕

    同一委员依次提交“通过、不通过、通过”，每次同时修改意见。

    通过标准：HR 可导出三个完整版本，包含旧值、新值、人员和时间。

  2. 投票期间结果隔离

    分别使用普通委员、非委员和管理员账号检查所有入口。

    通过标准：委员无法查看票数、比例、意见或他人身份。

  3. 未投人员精准催促

    制造已投、未投和发送失败三种状态，再执行一键催投。

    通过标准：只通知未投委员，并保留发送结果和失败原因。

  4. 手动和自动关闭

    在未投齐时手动关闭，再创建一场短截止时间的自动关闭测试。

    通过标准：关闭后任何客户端或接口都不能新增或修改投票。

  5. 保存与恢复承诺

    核对合同中的保存期限、备份频率、恢复目标和退订后的数据处理。

    通过标准：书面确认每日备份、恢复能力和完整批量导出。

建议决策路径

当前直接推进自研 H5；第三方应用和低代码平台仅作为条件明确的替代路径。

推进自研 H5

以现有需求说明书为验收基线，完成钉钉身份接入、投票闭环、审计历史、催投日志和备份恢复。

建议：作为当前正式实施路径立项

保留问卷星 POC

企业尊享版公开价为 ¥12,000/年。仅在希望减少开发投入时启动，并要求五项门槛全部通过且关键能力写入合同。

成本：公开价 ¥12,000/年；条件：关键能力全部得到证明

需要平台化管理时评估宜搭

如果单位未来还有多类内部流程需要业务人员持续搭建，可单独评估宜搭专业版。

条件：接受低代码实施、平台依赖和外部备份工作
